Advanced Features

Jouster provides several advanced features that allow you to customize its behavior and extend its functionality. This page covers these advanced features and how to use them.

Configuration

Jouster can be configured using a configuration file or command-line options. The configuration file is a JavaScript file that exports a configuration object.

Configuration File

By default, Jouster looks for a configuration file named jouster.config.js in the root of your project. You can specify a different configuration file using the --config option.

// jouster.config.js
module.exports = {
  testRunnerType: 'jest',
  issueManager: 'github',
  github: {
    owner: 'TSavo',
    repo: 'jouster',
    token: process.env.GITHUB_TOKEN
  },
  templates: {
    open: './templates/open.md',
    close: './templates/close.md',
    reopen: './templates/reopen.md'
  },
  hooks: {
    beforeRun: './hooks/before-run.js',
    afterRun: './hooks/after-run.js',
    beforeIssueCreate: './hooks/before-issue-create.js',
    afterIssueCreate: './hooks/after-issue-create.js',
    beforeIssueClose: './hooks/before-issue-close.js',
    afterIssueClose: './hooks/after-issue-close.js',
    beforeIssueReopen: './hooks/before-issue-reopen.js',
    afterIssueReopen: './hooks/after-issue-reopen.js'
  },
  plugins: [
    './plugins/custom-plugin.js'
  ]
};

Command-Line Options

You can also configure Jouster using command-line options. These options override the configuration in the configuration file.

npx jouster run-tests --test-runner jest --tracker-type github --github-owner TSavo --github-repo jouster

For a complete list of command-line options, see the Usage Guide.

Templates

Jouster uses templates to generate the content of issues. You can customize these templates to include additional information or format the content differently.

For detailed information about templates, see the Templates page.

Hooks

Hooks allow you to execute custom code at specific points in Jouster's execution. You can use hooks to customize Jouster's behavior or integrate it with other tools.

Available Hooks

Jouster provides the following hooks:

  • beforeRun: Executed before running tests.
  • afterRun: Executed after running tests.
  • beforeIssueCreate: Executed before creating an issue.
  • afterIssueCreate: Executed after creating an issue.
  • beforeIssueClose: Executed before closing an issue.
  • afterIssueClose: Executed after closing an issue.
  • beforeIssueReopen: Executed before reopening an issue.
  • afterIssueReopen: Executed after reopening an issue.

Hook Implementation

Hooks are implemented as JavaScript modules that export a function. The function receives a context object that contains information about the current execution.

// hooks/before-run.js
module.exports = function(context) {
  console.log('Running tests with Jouster...');
  console.log('Test runner:', context.config.testRunnerType);
  console.log('Issue manager:', context.config.issueManager);
  
  // You can modify the context object to affect Jouster's behavior
  context.config.testRunnerOptions.jest.bail = true;
  
  // You can also return a promise to perform asynchronous operations
  return Promise.resolve();
};

Hook Context

The context object passed to hooks contains the following properties:

  • config: The current configuration.
  • testRunner: The test runner instance.
  • issueManager: The issue manager instance.
  • mappingStore: The mapping store instance.
  • testResults: The test results (only available in afterRun and later hooks).
  • issue: The issue being created, closed, or reopened (only available in issue-related hooks).
  • test: The test associated with the issue (only available in issue-related hooks).

Plugins

Plugins allow you to extend Jouster's functionality with custom features. You can use plugins to add new test runners, issue managers, or other functionality.

Plugin Implementation

Plugins are implemented as JavaScript modules that export a function. The function receives a Jouster instance and can modify it to add new functionality.

// plugins/custom-plugin.js
module.exports = function(jouster) {
  // Add a new test runner
  jouster.addTestRunner('custom', require('./custom-test-runner'));
  
  // Add a new issue manager
  jouster.addIssueManager('custom', require('./custom-issue-manager'));
  
  // Add a new command
  jouster.addCommand('custom-command', require('./custom-command'));
  
  // Add a new hook
  jouster.addHook('customHook', require('./custom-hook'));
};

Plugin Registration

Plugins are registered in the configuration file using the plugins property. The property is an array of plugin module paths or plugin objects.

// jouster.config.js
module.exports = {
  // ... other configuration
  plugins: [
    './plugins/custom-plugin.js',
    require('./plugins/another-plugin')
  ]
};

Custom Test Runners

Jouster supports several test runners out of the box, but you can also create custom test runners to support other test frameworks.

Test Runner Implementation

A test runner is a JavaScript module that exports an object with the following methods:

  • run: Runs the tests and returns the results.
  • parseResults: Parses the test results into a standard format.
  • generateTestId: Generates a unique identifier for a test.
// custom-test-runner.js
module.exports = {
  run: async function(options) {
    // Run the tests and return the results
    const { spawn } = require('child_process');
    const child = spawn('custom-test-runner', ['--reporter', 'json']);
    
    let stdout = '';
    child.stdout.on('data', (data) => {
      stdout += data.toString();
    });
    
    return new Promise((resolve, reject) => {
      child.on('close', (code) => {
        if (code === 0) {
          resolve(stdout);
        } else {
          reject(new Error(`Test runner exited with code ${code}`));
        }
      });
    });
  },
  
  parseResults: function(results) {
    // Parse the results into a standard format
    const parsedResults = JSON.parse(results);
    
    return {
      passed: parsedResults.passed,
      failed: parsedResults.failed,
      tests: parsedResults.tests.map((test) => ({
        name: test.name,
        file: test.file,
        status: test.status,
        error: test.error
      }))
    };
  },
  
  generateTestId: function(test) {
    // Generate a unique identifier for the test
    return `${test.file}:${test.name}`;
  }
};

Test Runner Registration

Custom test runners are registered using the addTestRunner method in a plugin:

// plugins/custom-plugin.js
module.exports = function(jouster) {
  jouster.addTestRunner('custom', require('./custom-test-runner'));
};

Custom Issue Managers

Jouster supports GitHub and file-based issue managers out of the box, but you can also create custom issue managers to support other bug tracking systems.

Issue Manager Implementation

An issue manager is a JavaScript module that exports an object with the following methods:

  • createIssue: Creates a new issue.
  • closeIssue: Closes an issue.
  • reopenIssue: Reopens a closed issue.
  • getIssue: Gets information about an issue.
// custom-issue-manager.js
module.exports = {
  createIssue: async function(issue, options) {
    // Create a new issue in the bug tracking system
    const response = await fetch('https://api.example.com/issues', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${options.token}`
      },
      body: JSON.stringify({
        title: issue.title,
        body: issue.body,
        labels: issue.labels
      })
    });
    
    const data = await response.json();
    
    return {
      id: data.id,
      url: data.url
    };
  },
  
  closeIssue: async function(issueId, options) {
    // Close an issue in the bug tracking system
    await fetch(`https://api.example.com/issues/${issueId}`, {
      method: 'PATCH',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${options.token}`
      },
      body: JSON.stringify({
        state: 'closed'
      })
    });
  },
  
  reopenIssue: async function(issueId, options) {
    // Reopen a closed issue in the bug tracking system
    await fetch(`https://api.example.com/issues/${issueId}`, {
      method: 'PATCH',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${options.token}`
      },
      body: JSON.stringify({
        state: 'open'
      })
    });
  },
  
  getIssue: async function(issueId, options) {
    // Get information about an issue in the bug tracking system
    const response = await fetch(`https://api.example.com/issues/${issueId}`, {
      headers: {
        'Authorization': `Bearer ${options.token}`
      }
    });
    
    const data = await response.json();
    
    return {
      id: data.id,
      title: data.title,
      body: data.body,
      state: data.state,
      url: data.url
    };
  }
};

Issue Manager Registration

Custom issue managers are registered using the addIssueManager method in a plugin:

// plugins/custom-plugin.js
module.exports = function(jouster) {
  jouster.addIssueManager('custom', require('./custom-issue-manager'));
};
