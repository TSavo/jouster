Vitest Test Runner

Vitest is a blazing fast unit test framework powered by Vite. Jouster provides integration with Vitest, allowing you to track test failures and manage issues automatically.

Installation

To use Vitest with Jouster, you need to install Vitest:

npm install --save-dev vitest

Then, configure Jouster to use Vitest as the test runner:

// jouster.config.js
module.exports = {
  // ... other configuration
  testRunnerType: 'vitest'
};

Configuration

Jouster provides several configuration options for Vitest:

// jouster.config.js
module.exports = {
  // ... other configuration
  testRunnerType: 'vitest',
  testRunnerOptions: {
    vitest: {
      include: ['**/*.{test,spec}.{js,mjs,cjs,ts,mts,cts,jsx,tsx}'], // Files to include
      exclude: ['**/node_modules/**', '**/dist/**'], // Files to exclude
      watch: false, // Watch for changes
      ui: false, // Enable UI
      coverage: false, // Enable coverage
      update: false, // Update snapshots
      reporters: ['default'], // Reporters to use
      silent: false, // Disable console output
      environment: 'node', // Environment to run tests in
      threads: true, // Enable threading
      isolate: true, // Isolate environment for each test file
      root: '.', // Root directory
      testTimeout: 5000, // Test timeout in milliseconds
      hookTimeout: 5000, // Hook timeout in milliseconds
      bail: 0, // Number of failures before stopping
      retry: 0 // Number of retries
    }
  }
};

Usage Examples

Basic Usage

npx jouster run-tests --test-runner vitest

With Custom Include Pattern

npx jouster run-tests --test-runner vitest --vitest-include "**/*.spec.ts"

With Coverage

npx jouster run-tests --test-runner vitest --vitest-coverage

With Custom Environment

npx jouster run-tests --test-runner vitest --vitest-environment jsdom

With Custom Timeout

npx jouster run-tests --test-runner vitest --vitest-test-timeout 10000

Best Practices

1. Use Descriptive Test Names

Vitest uses test names for reporting, so use descriptive names that clearly indicate what the test is checking:

import { test, expect } from 'vitest';

test('sum() returns the sum of two numbers', () => {
  expect(sum(1, 2)).toBe(3);
});

2. Group Related Tests

Group related tests together to make the test output more organized:

import { describe, test, expect } from 'vitest';

describe('Calculator', () => {
  describe('sum()', () => {
    test('returns the sum of two positive numbers', () => {
      expect(sum(1, 2)).toBe(3);
    });
    
    test('returns the sum of a positive and a negative number', () => {
      expect(sum(1, -2)).toBe(-1);
    });
    
    test('returns the sum of two negative numbers', () => {
      expect(sum(-1, -2)).toBe(-3);
    });
  });
});

3. Use Matchers Effectively

Vitest provides a variety of matchers for different types of checks:

import { test, expect } from 'vitest';

test('matchers example', () => {
  expect(true).toBe(true);
  expect(false).not.toBe(true);
  expect(1).toEqual(1);
  expect([1, 2, 3]).toContain(2);
  expect({ a: 1, b: 2 }).toEqual({ a: 1, b: 2 });
  expect(null).toBeNull();
  expect(undefined).toBeUndefined();
  expect(1).toBeDefined();
  expect(0).toBeFalsy();
  expect(1).toBeTruthy();
  expect(1).toBeGreaterThan(0);
  expect(1).toBeLessThan(2);
  expect(() => { throw new Error('error'); }).toThrow();
  expect(() => { throw new Error('error'); }).toThrowError('error');
});

4. Use beforeEach and afterEach

Use beforeEach and afterEach to set up and tear down test fixtures:

import { describe, beforeEach, afterEach, test, expect } from 'vitest';

describe('Calculator', () => {
  let calculator;
  
  beforeEach(() => {
    calculator = new Calculator();
  });
  
  afterEach(() => {
    // Clean up if needed
  });
  
  test('sum() returns the sum of two numbers', () => {
    expect(calculator.sum(1, 2)).toBe(3);
  });
});

Troubleshooting

Common Issues

Tests Not Running

  • Check that Vitest is installed
  • Check that the test files exist
  • Check that the test files match the include pattern
  • Check that the test files have the correct extension

Tests Running but Not Reporting

  • Check that Vitest's output is being captured
  • Check that the silent option is not enabled

Tests Running but Not Failing

  • Check that expectations are being made
  • Check that the expectations are correct

Tests Running but Not Creating Issues

  • Check that issue tracking is enabled
  • Check that the GitHub client is configured correctly
  • Check that Vitest's test results are being parsed correctly
