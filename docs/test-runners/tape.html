Tape Test Runner

Tape is a TAP-producing test harness for Node.js with a minimal API. Jouster provides integration with Tape, allowing you to track test failures and manage issues automatically.

Installation

To use Tape with Jouster, you need to install Tape:

npm install --save-dev tape

Then, configure Jouster to use Tape as the test runner:

// jouster.config.js
module.exports = {
  // ... other configuration
  testRunnerType: 'tape'
};

Configuration

Jouster provides several configuration options for Tape:

// jouster.config.js
module.exports = {
  // ... other configuration
  testRunnerType: 'tape',
  testRunnerOptions: {
    tape: {
      files: ['test/**/*.js'], // Files to include
      timeout: 5000, // Test timeout in milliseconds
      bail: false, // Stop after first test failure
      color: true, // Enable colored output
      outputStream: process.stdout, // Stream to write output to
      harness: {}, // Options for the test harness
      createStream: {} // Options for the output stream
    }
  }
};

Usage Examples

Basic Usage

npx jouster run-tests --test-runner tape

With Custom Files

npx jouster run-tests --test-runner tape --tape-files test/**/*.js

With Custom Timeout

npx jouster run-tests --test-runner tape --tape-timeout 10000

With Bail Option

npx jouster run-tests --test-runner tape --tape-bail

With TypeScript

npx jouster run-tests --test-runner tape --tape-files test/**/*.ts --require ts-node/register

Best Practices

1. Use Descriptive Test Names

Tape uses test names for reporting, so use descriptive names that clearly indicate what the test is checking:

const test = require('tape');

test('sum() returns the sum of two numbers', t => {
  t.equal(sum(1, 2), 3);
  t.end();
});

2. Group Related Tests

Group related tests together to make the test output more organized:

const test = require('tape');

test('sum() function', t => {
  t.test('returns the sum of two positive numbers', st => {
    st.equal(sum(1, 2), 3);
    st.end();
  });
  
  t.test('returns the sum of a positive and a negative number', st => {
    st.equal(sum(1, -2), -1);
    st.end();
  });
  
  t.test('returns the sum of two negative numbers', st => {
    st.equal(sum(-1, -2), -3);
    st.end();
  });
  
  t.end();
});

3. Use Assertions Effectively

Tape provides a variety of assertions for different types of checks:

test('assertions example', t => {
  t.pass('this assertion passes');
  t.fail('this assertion fails');
  t.ok(true, 'this value is truthy');
  t.notOk(false, 'this value is falsy');
  t.equal(1, 1, 'these values are equal');
  t.notEqual(1, 2, 'these values are not equal');
  t.deepEqual({ a: 1 }, { a: 1 }, 'these objects are deeply equal');
  t.notDeepEqual({ a: 1 }, { a: 2 }, 'these objects are not deeply equal');
  t.throws(() => { throw new Error('error'); }, 'this function throws');
  t.doesNotThrow(() => {}, 'this function does not throw');
  t.end();
});

4. Always Call t.end()

Always call t.end() to signal the end of a test, or use t.plan(n) to specify the number of assertions:

test('using t.end()', t => {
  t.equal(sum(1, 2), 3);
  t.end();
});

test('using t.plan()', t => {
  t.plan(1);
  t.equal(sum(1, 2), 3);
});

Troubleshooting

Common Issues

Tests Not Running

  • Check that Tape is installed
  • Check that the test files exist
  • Check that the test files match the pattern
  • Check that the test files have the correct extension

Tests Running but Not Reporting

  • Check that Tape's output is being captured
  • Check that the output stream is configured correctly

Tests Running but Not Failing

  • Check that assertions are being made
  • Check that t.end() is being called

Tests Running but Not Creating Issues

  • Check that issue tracking is enabled
  • Check that the GitHub client is configured correctly
  • Check that Tape's test results are being parsed correctly
