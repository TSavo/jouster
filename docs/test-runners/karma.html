Karma Test Runner

Karma is a test runner that runs JavaScript tests in real browsers. Jouster provides integration with Karma, allowing you to track test failures and manage issues automatically.

Installation

To use Karma with Jouster, you need to install Karma and its dependencies:

npm install --save-dev karma karma-chrome-launcher karma-jasmine jasmine-core

Then, configure Jouster to use Karma as the test runner:

// jouster.config.js
module.exports = {
  // ... other configuration
  testRunnerType: 'karma'
};

Configuration

Jouster provides several configuration options for Karma:

// jouster.config.js
module.exports = {
  // ... other configuration
  testRunnerType: 'karma',
  testRunnerOptions: {
    karma: {
      browsers: ['Chrome'], // Browsers to run tests in
      configFile: 'karma.conf.js', // Path to Karma configuration file
      frameworks: ['jasmine'], // Test frameworks to use
      files: ['src/**/*.js', 'test/**/*.js'], // Files to include
      exclude: [], // Files to exclude
      preprocessors: {}, // Preprocessors to use
      reporters: ['progress'], // Reporters to use
      port: 9876, // Port to use
      colors: true, // Enable colors in reporter output
      logLevel: 'INFO', // Log level
      autoWatch: false, // Watch for file changes
      singleRun: true, // Run once and exit
      concurrency: Infinity, // Number of concurrent browser instances
      client: {
        clearContext: false // Leave Jasmine Spec Runner output visible in browser
      }
    }
  }
};

Usage Examples

Basic Usage

npx jouster run-tests --test-runner karma

With Custom Configuration File

npx jouster run-tests --test-runner karma --karma-config-file ./config/karma.conf.js

With Multiple Browsers

npx jouster run-tests --test-runner karma --karma-browsers "Chrome,Firefox"

With Custom Frameworks

npx jouster run-tests --test-runner karma --karma-frameworks "mocha,chai"

With Custom Reporters

npx jouster run-tests --test-runner karma --karma-reporters "progress,coverage"

Best Practices

1. Use Descriptive Test Names

Karma uses test names for reporting, so use descriptive names that clearly indicate what the test is checking:

describe('Calculator', function() {
  describe('sum()', function() {
    it('should return the sum of two numbers', function() {
      expect(sum(1, 2)).toBe(3);
    });
  });
});

2. Group Related Tests

Group related tests together to make the test output more organized:

describe('Calculator', function() {
  describe('sum()', function() {
    it('should return the sum of two positive numbers', function() {
      expect(sum(1, 2)).toBe(3);
    });
    
    it('should return the sum of a positive and a negative number', function() {
      expect(sum(1, -2)).toBe(-1);
    });
    
    it('should return the sum of two negative numbers', function() {
      expect(sum(-1, -2)).toBe(-3);
    });
  });
});

3. Use beforeEach and afterEach

Use beforeEach and afterEach to set up and tear down test fixtures:

describe('Calculator', function() {
  let calculator;
  
  beforeEach(function() {
    calculator = new Calculator();
  });
  
  afterEach(function() {
    // Clean up if needed
  });
  
  describe('sum()', function() {
    it('should return the sum of two numbers', function() {
      expect(calculator.sum(1, 2)).toBe(3);
    });
  });
});

4. Use Spies for Mocking

Use spies to mock functions and verify they were called:

describe('Logger', function() {
  let logger;
  
  beforeEach(function() {
    logger = new Logger();
    spyOn(console, 'log');
  });
  
  it('should log a message', function() {
    logger.log('Hello, world!');
    expect(console.log).toHaveBeenCalledWith('Hello, world!');
  });
});

Troubleshooting

Common Issues

Tests Not Running

  • Check that Karma is installed
  • Check that the browsers are installed
  • Check that the test files exist
  • Check that the test files match the pattern

Tests Running but Not Reporting

  • Check that Karma's output is being captured
  • Check that the reporter is configured correctly

Tests Running but Not Failing

  • Check that assertions are being made
  • Check that the assertions are correct

Tests Running but Not Creating Issues

  • Check that issue tracking is enabled
  • Check that the GitHub client is configured correctly
  • Check that Karma's test results are being parsed correctly
