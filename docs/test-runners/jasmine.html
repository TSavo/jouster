Jasmine Test Runner

Jasmine is a behavior-driven development framework for testing JavaScript code. Jouster provides integration with Jasmine, allowing you to track test failures and manage issues automatically.

Installation

To use Jasmine with Jouster, you need to install Jasmine:

npm install --save-dev jasmine

Then, configure Jouster to use Jasmine as the test runner:

// jouster.config.js
module.exports = {
  // ... other configuration
  testRunnerType: 'jasmine'
};

Configuration

Jouster provides several configuration options for Jasmine:

// jouster.config.js
module.exports = {
  // ... other configuration
  testRunnerType: 'jasmine',
  testRunnerOptions: {
    jasmine: {
      spec_dir: 'spec', // Directory with spec files
      spec_files: ['**/*[sS]pec.js'], // Files to include
      helpers: ['helpers/**/*.js'], // Helper files
      stopSpecOnExpectationFailure: false, // Stop spec on first failure
      random: true, // Run specs in random order
      seed: null, // Seed for random order
      failFast: false, // Stop execution after first spec failure
      color: true, // Enable colored output
      reporter: null, // Custom reporter
      timeout: 5000 // Default timeout in milliseconds
    }
  }
};

Usage Examples

Basic Usage

npx jouster run-tests --test-runner jasmine

With Custom Spec Directory

npx jouster run-tests --test-runner jasmine --jasmine-spec-dir test

With Custom Spec Files

npx jouster run-tests --test-runner jasmine --jasmine-spec-files "**/*.spec.js"

With Custom Timeout

npx jouster run-tests --test-runner jasmine --jasmine-timeout 10000

With TypeScript

npx jouster run-tests --test-runner jasmine --jasmine-helpers "helpers/ts-node.js"

Where helpers/ts-node.js contains:

require('ts-node').register({
  transpileOnly: true
});

Best Practices

1. Use Descriptive Spec Names

Jasmine uses spec names for reporting, so use descriptive names that clearly indicate what the spec is checking:

describe('Calculator', () => {
  describe('sum()', () => {
    it('returns the sum of two numbers', () => {
      expect(sum(1, 2)).toBe(3);
    });
  });
});

2. Group Related Specs

Group related specs together to make the test output more organized:

describe('Calculator', () => {
  describe('sum()', () => {
    it('returns the sum of two positive numbers', () => {
      expect(sum(1, 2)).toBe(3);
    });
    
    it('returns the sum of a positive and a negative number', () => {
      expect(sum(1, -2)).toBe(-1);
    });
    
    it('returns the sum of two negative numbers', () => {
      expect(sum(-1, -2)).toBe(-3);
    });
  });
});

3. Use Matchers Effectively

Jasmine provides a variety of matchers for different types of checks:

describe('Matchers example', () => {
  it('demonstrates various matchers', () => {
    expect(true).toBe(true);
    expect(false).not.toBe(true);
    expect(1).toEqual(1);
    expect([1, 2, 3]).toContain(2);
    expect({ a: 1, b: 2 }).toEqual({ a: 1, b: 2 });
    expect(null).toBeNull();
    expect(undefined).toBeUndefined();
    expect(1).toBeDefined();
    expect(0).toBeFalsy();
    expect(1).toBeTruthy();
    expect(1).toBeGreaterThan(0);
    expect(1).toBeLessThan(2);
    expect(() => { throw new Error('error'); }).toThrow();
    expect(() => { throw new Error('error'); }).toThrowError('error');
  });
});

4. Use beforeEach and afterEach

Use beforeEach and afterEach to set up and tear down test fixtures:

describe('Calculator', () => {
  let calculator;
  
  beforeEach(() => {
    calculator = new Calculator();
  });
  
  afterEach(() => {
    // Clean up if needed
  });
  
  describe('sum()', () => {
    it('returns the sum of two numbers', () => {
      expect(calculator.sum(1, 2)).toBe(3);
    });
  });
});

Troubleshooting

Common Issues

Specs Not Running

  • Check that Jasmine is installed
  • Check that the spec files exist
  • Check that the spec files match the pattern
  • Check that the spec files have the correct extension

Specs Running but Not Reporting

  • Check that Jasmine's output is being captured
  • Check that the reporter is configured correctly

Specs Running but Not Failing

  • Check that expectations are being made
  • Check that the expectations are correct

Specs Running but Not Creating Issues

  • Check that issue tracking is enabled
  • Check that the GitHub client is configured correctly
  • Check that Jasmine's spec results are being parsed correctly
