AVA Test Runner

AVA is a test runner for Node.js with a concise API, detailed error output, and process isolation. Jouster provides integration with AVA, allowing you to track test failures and manage issues automatically.

Installation

To use AVA with Jouster, you need to install AVA:

npm install --save-dev ava

Then, configure Jouster to use AVA as the test runner:

// jouster.config.js
module.exports = {
  // ... other configuration
  testRunnerType: 'ava'
};

Configuration

Jouster provides several configuration options for AVA:

// jouster.config.js
module.exports = {
  // ... other configuration
  testRunnerType: 'ava',
  testRunnerOptions: {
    ava: {
      files: ['test/**/*.js'], // Files to include
      match: [], // Patterns to match test titles
      timeout: '10s', // Test timeout
      concurrency: 5, // Number of test files to run at the same time
      failFast: false, // Stop after first test failure
      failWithoutAssertions: true, // Fail test if it doesn't run assertions
      environmentVariables: {}, // Environment variables to set
      require: [], // Modules to require before running tests
      verbose: false, // Enable verbose output
      serial: false, // Run tests serially
      color: true, // Enable colored output
      watch: false // Watch files for changes
    }
  }
};

Usage Examples

Basic Usage

npx jouster run-tests --test-runner ava

With Custom Files

npx jouster run-tests --test-runner ava --ava-files test/**/*.js

With Custom Timeout

npx jouster run-tests --test-runner ava --ava-timeout 20s

With Custom Concurrency

npx jouster run-tests --test-runner ava --ava-concurrency 10

With TypeScript

npx jouster run-tests --test-runner ava --ava-require ts-node/register

Best Practices

1. Use Descriptive Test Titles

AVA uses test titles for reporting, so use descriptive titles that clearly indicate what the test is checking:

test('sum() returns the sum of two numbers', t => {
  t.is(sum(1, 2), 3);
});

2. Use Test Macros for Similar Tests

AVA supports test macros for running similar tests with different inputs:

const macro = (t, input, expected) => {
  t.is(sum(input.a, input.b), expected);
};

test('sum() returns the sum of two positive numbers', macro, { a: 1, b: 2 }, 3);
test('sum() returns the sum of a positive and a negative number', macro, { a: 1, b: -2 }, -1);
test('sum() returns the sum of two negative numbers', macro, { a: -1, b: -2 }, -3);

3. Use Assertions Effectively

AVA provides a variety of assertions for different types of checks:

test('assertions example', t => {
  t.pass('this assertion passes');
  t.fail('this assertion fails');
  t.truthy('this value is truthy');
  t.falsy('');
  t.true(true);
  t.false(false);
  t.is(1, 1);
  t.not(1, 2);
  t.deepEqual({ a: 1 }, { a: 1 });
  t.notDeepEqual({ a: 1 }, { a: 2 });
  t.throws(() => { throw new Error('error'); });
  t.notThrows(() => {});
  t.regex('abc', /^a/);
  t.notRegex('abc', /^b/);
});

4. Use Async/Await for Asynchronous Tests

AVA supports async/await for asynchronous tests:

test('async test', async t => {
  const result = await someAsyncFunction();
  t.is(result, 'expected value');
});

Troubleshooting

Common Issues

Tests Not Running

  • Check that AVA is installed
  • Check that the test files exist
  • Check that the test files match the pattern
  • Check that the test files have the correct extension

Tests Running but Not Reporting

  • Check that AVA's output is being captured
  • Check that the verbose option is enabled

Tests Running but Not Failing

  • Check that assertions are being made
  • Check that the failWithoutAssertions option is enabled

Tests Running but Not Creating Issues

  • Check that issue tracking is enabled
  • Check that the GitHub client is configured correctly
  • Check that AVA's test results are being parsed correctly
