Cypress Test Runner

Cypress is a next-generation front end testing tool built for the modern web. Jouster provides integration with Cypress, allowing you to track test failures and manage issues automatically.

Installation

To use Cypress with Jouster, you need to install Cypress:

npm install --save-dev cypress

Then, configure Jouster to use Cypress as the test runner:

// jouster.config.js
module.exports = {
  // ... other configuration
  testRunnerType: 'cypress'
};

Configuration

Jouster provides several configuration options for Cypress:

// jouster.config.js
module.exports = {
  // ... other configuration
  testRunnerType: 'cypress',
  testRunnerOptions: {
    cypress: {
      browser: 'chrome', // Browser to run tests in
      config: {}, // Cypress configuration
      configFile: 'cypress.json', // Path to Cypress configuration file
      env: {}, // Environment variables
      headed: false, // Run in headed mode
      headless: true, // Run in headless mode
      project: '.', // Path to project
      quiet: false, // Suppress output
      record: false, // Record results to Cypress Dashboard
      reporter: 'spec', // Reporter to use
      reporterOptions: {}, // Reporter options
      spec: '**/*.cy.{js,jsx,ts,tsx}', // Spec files to run
      testingType: 'e2e' // Testing type (e2e or component)
    }
  }
};

Usage Examples

Basic Usage

npx jouster run-tests --test-runner cypress

With Custom Spec Pattern

npx jouster run-tests --test-runner cypress --cypress-spec "cypress/e2e/**/*.cy.js"

With Headed Mode

npx jouster run-tests --test-runner cypress --cypress-headed

With Custom Browser

npx jouster run-tests --test-runner cypress --cypress-browser firefox

With Component Testing

npx jouster run-tests --test-runner cypress --cypress-testing-type component

Best Practices

1. Use Descriptive Test Names

Cypress uses test names for reporting, so use descriptive names that clearly indicate what the test is checking:

describe('Login Form', () => {
  it('should log in with valid credentials', () => {
    cy.visit('/login');
    cy.get('#username').type('testuser');
    cy.get('#password').type('password123');
    cy.get('button[type="submit"]').click();
    cy.url().should('include', '/dashboard');
  });
});

2. Group Related Tests

Group related tests together to make the test output more organized:

describe('Login Form', () => {
  beforeEach(() => {
    cy.visit('/login');
  });
  
  it('should log in with valid credentials', () => {
    cy.get('#username').type('testuser');
    cy.get('#password').type('password123');
    cy.get('button[type="submit"]').click();
    cy.url().should('include', '/dashboard');
  });
  
  it('should show error with invalid credentials', () => {
    cy.get('#username').type('testuser');
    cy.get('#password').type('wrongpassword');
    cy.get('button[type="submit"]').click();
    cy.get('.error-message').should('be.visible');
  });
  
  it('should validate required fields', () => {
    cy.get('button[type="submit"]').click();
    cy.get('#username:invalid').should('exist');
    cy.get('#password:invalid').should('exist');
  });
});

3. Use Custom Commands

Use custom commands to encapsulate common actions:

// cypress/support/commands.js
Cypress.Commands.add('login', (username, password) => {
  cy.visit('/login');
  cy.get('#username').type(username);
  cy.get('#password').type(password);
  cy.get('button[type="submit"]').click();
});

// cypress/e2e/login.cy.js
describe('Login Form', () => {
  it('should log in with valid credentials', () => {
    cy.login('testuser', 'password123');
    cy.url().should('include', '/dashboard');
  });
});

4. Use Fixtures for Test Data

Use fixtures to store test data:

// cypress/fixtures/users.json
{
  "validUser": {
    "username": "testuser",
    "password": "password123"
  },
  "invalidUser": {
    "username": "testuser",
    "password": "wrongpassword"
  }
}

// cypress/e2e/login.cy.js
describe('Login Form', () => {
  beforeEach(() => {
    cy.fixture('users').as('users');
  });
  
  it('should log in with valid credentials', function() {
    cy.visit('/login');
    cy.get('#username').type(this.users.validUser.username);
    cy.get('#password').type(this.users.validUser.password);
    cy.get('button[type="submit"]').click();
    cy.url().should('include', '/dashboard');
  });
});

Troubleshooting

Common Issues

Tests Not Running

  • Check that Cypress is installed
  • Check that the spec files exist
  • Check that the spec files match the pattern
  • Check that the spec files have the correct extension

Tests Running but Not Reporting

  • Check that Cypress's output is being captured
  • Check that the reporter is configured correctly

Tests Running but Not Failing

  • Check that assertions are being made
  • Check that the assertions are correct

Tests Running but Not Creating Issues

  • Check that issue tracking is enabled
  • Check that the GitHub client is configured correctly
  • Check that Cypress's test results are being parsed correctly
