TestCafe Test Runner

TestCafe is a Node.js tool to automate end-to-end web testing. Jouster provides integration with TestCafe, allowing you to track test failures and manage issues automatically.

Installation

To use TestCafe with Jouster, you need to install TestCafe:

npm install --save-dev testcafe

Then, configure Jouster to use TestCafe as the test runner:

// jouster.config.js
module.exports = {
  // ... other configuration
  testRunnerType: 'testcafe'
};

Configuration

Jouster provides several configuration options for TestCafe:

// jouster.config.js
module.exports = {
  // ... other configuration
  testRunnerType: 'testcafe',
  testRunnerOptions: {
    testcafe: {
      browsers: ['chrome'], // Browsers to run tests in
      concurrency: 1, // Number of concurrent browser instances
      filter: null, // Filter tests by name
      fixture: null, // Filter tests by fixture name
      reporter: 'spec', // Reporter to use
      reporterOptions: {}, // Reporter options
      screenshots: {
        path: 'screenshots', // Path to save screenshots
        takeOnFails: true // Take screenshots on test failures
      },
      skipJsErrors: false, // Skip JavaScript errors
      skipUncaughtErrors: false, // Skip uncaught errors
      src: ['tests/**/*.js'], // Test files
      speed: 1, // Test execution speed (0.01 to 1)
      stopOnFirstFail: false, // Stop after first test failure
      testTimeout: 30000 // Test timeout in milliseconds
    }
  }
};

Usage Examples

Basic Usage

npx jouster run-tests --test-runner testcafe

With Custom Test Files

npx jouster run-tests --test-runner testcafe --testcafe-src "e2e/**/*.test.js"

With Multiple Browsers

npx jouster run-tests --test-runner testcafe --testcafe-browsers "chrome,firefox"

With Concurrency

npx jouster run-tests --test-runner testcafe --testcafe-concurrency 4

With Test Filter

npx jouster run-tests --test-runner testcafe --testcafe-filter "Login tests"

Best Practices

1. Use Descriptive Test Names

TestCafe uses test names for reporting, so use descriptive names that clearly indicate what the test is checking:

import { Selector } from 'testcafe';

fixture('Login Form')
  .page('http://example.com/login');

test('should log in with valid credentials', async t => {
  await t
    .typeText('#username', 'testuser')
    .typeText('#password', 'password123')
    .click('button[type="submit"]')
    .expect(Selector('h1').withText('Dashboard').exists).ok();
});

2. Group Related Tests

Group related tests together to make the test output more organized:

import { Selector } from 'testcafe';

fixture('Login Form')
  .page('http://example.com/login');

test('should log in with valid credentials', async t => {
  await t
    .typeText('#username', 'testuser')
    .typeText('#password', 'password123')
    .click('button[type="submit"]')
    .expect(Selector('h1').withText('Dashboard').exists).ok();
});

test('should show error with invalid credentials', async t => {
  await t
    .typeText('#username', 'testuser')
    .typeText('#password', 'wrongpassword')
    .click('button[type="submit"]')
    .expect(Selector('.error-message').exists).ok();
});

test('should validate required fields', async t => {
  await t
    .click('button[type="submit"]')
    .expect(Selector('#username').hasAttribute('required')).ok()
    .expect(Selector('#password').hasAttribute('required')).ok();
});

3. Use Page Object Model

Use the Page Object Model pattern to encapsulate page interactions:

// models/LoginPage.js
import { Selector, t } from 'testcafe';

export class LoginPage {
  constructor() {
    this.usernameInput = Selector('#username');
    this.passwordInput = Selector('#password');
    this.submitButton = Selector('button[type="submit"]');
    this.errorMessage = Selector('.error-message');
    this.dashboardHeader = Selector('h1').withText('Dashboard');
  }
  
  async login(username, password) {
    await t
      .typeText(this.usernameInput, username)
      .typeText(this.passwordInput, password)
      .click(this.submitButton);
  }
}

// tests/login.test.js
import { LoginPage } from '../models/LoginPage';

const loginPage = new LoginPage();

fixture('Login Form')
  .page('http://example.com/login');

test('should log in with valid credentials', async t => {
  await loginPage.login('testuser', 'password123');
  await t.expect(loginPage.dashboardHeader.exists).ok();
});

4. Use Roles for Authentication

Use TestCafe's Role feature for authentication:

import { Selector, Role } from 'testcafe';

const loginPage = {
  usernameInput: Selector('#username'),
  passwordInput: Selector('#password'),
  submitButton: Selector('button[type="submit"]')
};

const regularUser = Role('http://example.com/login', async t => {
  await t
    .typeText(loginPage.usernameInput, 'testuser')
    .typeText(loginPage.passwordInput, 'password123')
    .click(loginPage.submitButton);
}, { preserveUrl: true });

const adminUser = Role('http://example.com/login', async t => {
  await t
    .typeText(loginPage.usernameInput, 'admin')
    .typeText(loginPage.passwordInput, 'adminpass')
    .click(loginPage.submitButton);
}, { preserveUrl: true });

fixture('User Dashboard')
  .page('http://example.com');

test('regular user can view dashboard', async t => {
  await t
    .useRole(regularUser)
    .navigateTo('/dashboard')
    .expect(Selector('h1').withText('Dashboard').exists).ok();
});

test('admin user can view admin panel', async t => {
  await t
    .useRole(adminUser)
    .navigateTo('/admin')
    .expect(Selector('h1').withText('Admin Panel').exists).ok();
});

Troubleshooting

Common Issues

Tests Not Running

  • Check that TestCafe is installed
  • Check that the test files exist
  • Check that the test files match the pattern
  • Check that the browsers are installed

Tests Running but Not Reporting

  • Check that TestCafe's output is being captured
  • Check that the reporter is configured correctly

Tests Running but Not Failing

  • Check that assertions are being made
  • Check that the assertions are correct

Tests Running but Not Creating Issues

  • Check that issue tracking is enabled
  • Check that the GitHub client is configured correctly
  • Check that TestCafe's test results are being parsed correctly
