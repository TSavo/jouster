Templates

Jouster uses templates to generate the content of issues. You can customize these templates to include additional information or format the content differently. This page explains how to use and customize templates in Jouster.

Template Types

Jouster uses three types of templates:

  • Open Template: Used when creating a new issue for a failing test.
  • Close Template: Used when closing an issue for a test that is now passing.
  • Reopen Template: Used when reopening an issue for a test that is failing again after it was fixed.

Each template type serves a different purpose and can be customized independently.

Default Templates

Jouster provides default templates for each template type. These templates include basic information about the test failure and are designed to be useful out of the box.

Custom Templates

You can customize the templates to include additional information or format the content differently. Custom templates are specified in the configuration file using the templates property.

Template Files

Custom templates are stored in files and referenced in the configuration file. The files can be in any format, but Markdown is recommended for GitHub issues.

// jouster.config.js
module.exports = {
  // ... other configuration
  templates: {
    open: './templates/open.md',
    close: './templates/close.md',
    reopen: './templates/reopen.md'
  }
};

Inline Templates

You can also specify templates inline in the configuration file:

module.exports = {

  templates: {

Template Variables

Templates can include variables that are replaced with actual values when the template is rendered. Variables are enclosed in double curly braces ({{variable}}).

Available Variables

The following variables are available in templates:

Test Variables

  • {{test.name}}: The name of the test.
  • {{test.file}}: The file containing the test.
  • {{test.line}}: The line number of the test.
  • {{test.column}}: The column number of the test.
  • {{test.error}}: The error message from the test failure.
  • {{test.stackTrace}}: The stack trace from the test failure.
  • {{test.duration}}: The duration of the test in milliseconds.

Test Runner Variables

  • {{testRunner.type}}: The type of test runner (Jest, Mocha, etc.).
  • {{testRunner.version}}: The version of the test runner.
  • {{testRunner.command}}: The command used to run the tests.

System Variables

  • {{system.os}}: The operating system.
  • {{system.nodeVersion}}: The version of Node.js.
  • {{system.jousterVersion}}: The version of Jouster.
  • {{system.date}}: The current date and time.
  • {{system.user}}: The current user.

Issue Variables

  • {{issue.id}}: The ID of the issue.
  • {{issue.url}}: The URL of the issue.
  • {{issue.title}}: The title of the issue.
  • {{issue.body}}: The body of the issue.
  • {{issue.state}}: The state of the issue (open, closed, etc.).
  • {{issue.labels}}: The labels of the issue.
  • {{issue.assignee}}: The assignee of the issue.
  • {{issue.createdAt}}: The creation date of the issue.
  • {{issue.updatedAt}}: The last update date of the issue.

Template Helpers

Jouster provides several helpers that can be used in templates to format data or perform operations.

Conditional Helpers

Conditional helpers allow you to include content based on conditions:

{{#if test.error}}
## Error
```
{{test.error}}
```
{{/if}}

{{#unless test.error}}
No error was reported.
{{/unless}}

Iteration Helpers

Iteration helpers allow you to iterate over arrays:

## Labels
{{#each issue.labels}}
- {{this}}
{{/each}}

Formatting Helpers

Formatting helpers allow you to format data:

## Created At
{{formatDate issue.createdAt "YYYY-MM-DD HH:mm:ss"}}

## Duration
{{formatDuration test.duration}}

## Stack Trace
{{formatStackTrace test.stackTrace}}

Template Best Practices

Include Essential Information

Make sure your templates include all the essential information needed to understand and fix the test failure:

  • Test name and location
  • Error message and stack trace
  • Test runner information
  • System information

Use Markdown Formatting

Use Markdown formatting to make your templates more readable:

  • Use headings for sections
  • Use code blocks for error messages and stack traces
  • Use lists for related information
  • Use bold or italic text for emphasis

Use Conditional Sections

Use conditional sections to include or exclude information based on its availability:

  • Use {{#if variable}} to include a section only if the variable is available
  • Use {{#unless variable}} to include a section only if the variable is not available
  • Use {{else}} to provide alternative content

Use Formatting Helpers

Use formatting helpers to format data in a more readable way:

  • Use {{formatDate}} to format dates
  • Use {{formatDuration}} to format durations
  • Use {{formatStackTrace}} to format stack traces
