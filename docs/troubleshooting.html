Troubleshooting

This page provides solutions to common issues you might encounter when using Jouster. If you're having trouble with Jouster, check this page for solutions to common problems.

Installation Issues

npm install fails

If you're having trouble installing Jouster with npm, try the following:

  • Make sure you have the latest version of npm installed: npm install -g npm@latest
  • Clear the npm cache: npm cache clean --force
  • Try installing with the --no-optional flag: npm install --no-optional jouster
  • Check your network connection and proxy settings

Peer dependency warnings

Jouster has peer dependencies on various test runners. If you see warnings about missing peer dependencies, you can install them manually:

npm install --save-dev jest mocha

Or you can ignore the warnings if you're not using those test runners.

Node.js version compatibility

Jouster requires Node.js version 14 or later. If you're using an older version of Node.js, you'll need to upgrade:

nvm install 14
nvm use 14

Or download the latest version from nodejs.org.

Configuration Issues

Configuration file not found

If Jouster can't find your configuration file, make sure it's in the right location and has the right name:

  • The default configuration file name is jouster.config.js
  • The configuration file should be in the root of your project
  • You can specify a different configuration file using the --config option: npx jouster run-tests --config ./path/to/config.js

Invalid configuration

If your configuration file is invalid, Jouster will show an error message. Make sure your configuration file is valid JavaScript and exports an object:

// jouster.config.js
module.exports = {
  testRunnerType: 'jest',
  issueManager: 'github',
  github: {
    owner: 'TSavo',
    repo: 'jouster',
    token: process.env.GITHUB_TOKEN
  }
};

Environment variables not set

If you're using environment variables in your configuration file, make sure they're set:

# Linux/macOS
export GITHUB_TOKEN=your-token

# Windows
set GITHUB_TOKEN=your-token

You can also use a .env file with dotenv:

// jouster.config.js
require('dotenv').config();

module.exports = {
  // ... configuration
};

Test Runner Issues

Test runner not found

If Jouster can't find your test runner, make sure it's installed and configured correctly:

  • Install the test runner: npm install --save-dev jest
  • Configure Jouster to use the test runner: testRunnerType: 'jest'
  • Make sure the test runner is in your PATH

Tests not running

If your tests aren't running, check the following:

  • Make sure your test files exist and match the pattern specified in your configuration
  • Check that your test runner is configured correctly
  • Try running the tests directly with the test runner: npx jest

Tests running but not failing

If your tests are running but not failing when they should, check the following:

  • Make sure your assertions are correct
  • Check that your test runner is configured to fail on errors
  • Try running the tests directly with the test runner: npx jest

Issue Manager Issues

GitHub authentication failed

If you're having trouble authenticating with GitHub, check the following:

  • Make sure your GitHub token is set correctly
  • Check that your token has the necessary permissions (repo, issues)
  • Verify that your token is not expired
  • Try creating a new token: GitHub Tokens

Issues not being created

If issues aren't being created for failing tests, check the following:

  • Make sure your issue manager is configured correctly
  • Check that your GitHub token has the necessary permissions
  • Verify that the repository exists and you have access to it
  • Check the mapping database to see if the test is already mapped to an issue

Issues not being closed

If issues aren't being closed for passing tests, check the following:

  • Make sure your issue manager is configured correctly
  • Check that your GitHub token has the necessary permissions
  • Verify that the issue exists and is open
  • Check the mapping database to see if the test is mapped to the correct issue

Mapping Database Issues

Mapping database not found

If Jouster can't find your mapping database, make sure it's in the right location and has the right name:

  • The default mapping database file name is test-issue-mapping.json
  • The mapping database file should be in the root of your project
  • You can specify a different mapping database file using the --mapping-file option: npx jouster run-tests --mapping-file ./path/to/mapping.json

Mapping database corrupted

If your mapping database is corrupted, you can try to fix it or create a new one:

  • Try to fix the JSON file manually
  • Create a new mapping database file: echo "{}" > test-issue-mapping.json
  • Run Jouster with the --clean-database option: npx jouster clean-database

Duplicate test identifiers

If you have duplicate test identifiers, Jouster might not be able to map tests to issues correctly. To fix this:

  • Make sure your test names are unique
  • Check that your test runner is generating unique identifiers for tests
  • Clean the mapping database: npx jouster clean-database

Template Issues

Template not found

If Jouster can't find your template, make sure it's in the right location and has the right name:

  • Check that the template file exists
  • Make sure the path to the template is correct in your configuration
  • You can specify a different template using the --template-open, --template-close, and --template-reopen options

Template variables not working

If template variables aren't being replaced with actual values, check the following:

  • Make sure the variable names are correct: {{test.name}}, {{test.error}}, etc.
  • Check that the variables are available in the context
  • Try using a simpler template to see if it works

Template helpers not working

If template helpers aren't working, check the following:

  • Make sure the helper names are correct: {{#if test.error}}, {{formatDate issue.createdAt}}, etc.
  • Check that the helpers are registered
  • Try using a simpler template to see if it works

Common Error Messages

Error: Cannot find module 'jouster'

This error means that Jouster is not installed or not in your node_modules directory. To fix it:

npm install --save-dev jouster

Error: Cannot find module 'jest'

This error means that Jest is not installed or not in your node_modules directory. To fix it:

npm install --save-dev jest

Error: Cannot find configuration file

This error means that Jouster can't find your configuration file. To fix it:

npx jouster init

This will create a default configuration file in the root of your project.

Error: GitHub API rate limit exceeded

This error means that you've exceeded GitHub's API rate limit. To fix it:

  • Wait for the rate limit to reset (usually 1 hour)
  • Use a GitHub token with higher rate limits
  • Reduce the number of API calls you're making

Error: Permission denied

This error means that you don't have permission to access a file or directory. To fix it:

  • Check the file permissions: ls -la
  • Change the file permissions: chmod 644 file.js
  • Run Jouster with elevated privileges (not recommended): sudo npx jouster
