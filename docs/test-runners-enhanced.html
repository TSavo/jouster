Test Runners

Jouster supports multiple test runners, allowing you to use your preferred testing framework while still benefiting from Jouster's issue tracking capabilities. This document provides an overview of the supported test runners and common configuration options.

Note: Pages marked with Enhanced include interactive demos and additional features.

Common Configuration

All test runners share some common configuration options:

module.exports = {
  // ... other configuration
  testRunnerType: 'auto', // 'jest', 'mocha', 'ava', 'tape', 'jasmine', 'vitest', 'cypress', 'playwright', 'testcafe', 'karma', 'generic', or 'auto'
  testRunnerOptions: {
    // Test runner-specific options
  }
};

The testRunnerType option specifies which test runner to use. If set to 'auto', Jouster will attempt to detect the test runner based on your project's dependencies.

Interactive Demo: Auto-Detection

Try out this interactive demo to see how Jouster's auto-detection works:

// Example of auto-detecting test runners
function detectTestRunner(dependencies) {
  const testRunners = [
    'jest',
    'mocha',
    'ava',
    'tape',
    'jasmine',
    'vitest',
    'cypress',
    '@playwright/test',
    'testcafe',
    'karma'
  ];
  
  // Find the first test runner in the dependencies
  const detectedRunner = testRunners.find(runner => 
    dependencies.hasOwnProperty(runner));
  
  // Return the detected runner or fall back to Jest
  return detectedRunner || 'jest';
}

// Test with different dependency sets
const results = [
  detectTestRunner({ 'mocha': '^9.0.0', 'chai': '^4.0.0' }),
  detectTestRunner({ '@playwright/test': '^1.0.0' }),
  detectTestRunner({ 'some-other-package': '^1.0.0' })
];

return {
  'With Mocha': results[0],
  'With Playwright': results[1],
  'With no test runners': results[2]
};
Click "Run" to see the result

Interactive Demo: Test Runner Configuration

Try out this interactive demo to see how to configure different test runners:

// Example of configuring different test runners
function createConfig(testRunner, options = {}) {
  return {
    testRunnerType: testRunner,
    testRunnerOptions: {
      [testRunner]: options
    }
  };
}

// Create configurations for different test runners
const jestConfig = createConfig('jest', {
  coverage: true,
  verbose: true,
  testTimeout: 10000
});

const mochaConfig = createConfig('mocha', {
  ui: 'bdd',
  timeout: 5000,
  reporters: ['spec'],
  require: ['ts-node/register']
});

const playwrightConfig = createConfig('playwright', {
  browser: 'chromium',
  headed: false,
  workers: 4,
  timeout: 30000
});

return {
  jest: jestConfig,
  mocha: mochaConfig,
  playwright: playwrightConfig
};
Click "Run" to see the result

Best Practices

1. Use the Right Test Runner for the Job

  • Use Jest for unit and integration tests in JavaScript/TypeScript projects
  • Use Mocha for flexible, customizable testing
  • Use AVA for concurrent test execution
  • Use Tape for simple, TAP-producing tests
  • Use Jasmine for behavior-driven development
  • Use Vitest for Vite-based projects
  • Use Cypress for end-to-end testing of web applications
  • Use Playwright for cross-browser end-to-end testing
  • Use TestCafe for end-to-end testing without browser plugins
  • Use Karma for testing in real browsers

2. Configure for Your Environment

  • Use headless browsers in CI environments
  • Use headed browsers for local development
  • Adjust concurrency based on available resources
  • Use appropriate timeouts for your tests

3. Use Consistent Reporting

  • Use the same reporter across all test runners
  • Configure reporters to provide the information you need
  • Use JSON reporters for machine-readable output
