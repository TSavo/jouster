Jouster Setup Wizard

The Jouster Setup Wizard is an interactive tool that helps you configure Jouster for your project. It guides you through the process of setting up Jouster with your preferred test runner and GitHub repository.

Running the Setup Wizard

To run the Setup Wizard, use the following command:

npx jouster setup

This will start an interactive wizard that will guide you through the setup process.

$ npx jouster setup

Welcome to the Jouster Setup Wizard!

This wizard will help you configure Jouster for your project.

? What test runner are you using? (Use arrow keys)
❯ Jest
  Mocha
  AVA
  Tape
  Jasmine
  Vitest
  Cypress

Setup Wizard Steps

Choose a Test Runner

The first step is to choose the test runner you're using in your project. Jouster supports the following test runners:

  • Jest
  • Mocha
  • AVA
  • Tape
  • Jasmine
  • Vitest
  • Cypress
  • Playwright
  • TestCafe
  • Karma
  • Generic (any command-line test tool)

The wizard will detect installed test runners in your project and suggest the most appropriate one.

Configure Test Runner Options

Next, you'll configure options specific to your chosen test runner. For example, if you chose Jest, you might configure:

  • Test pattern (e.g., **/*.test.js)
  • Test timeout
  • Whether to collect coverage information
  • Additional Jest options
? What test pattern should Jouster use? **/*.test.js
? What test timeout should Jouster use (in milliseconds)? 5000
? Should Jouster collect coverage information? Yes

Choose an Issue Tracker

Jouster can create issues in GitHub or in a local file system. Choose the issue tracker you want to use:

  • GitHub (requires GitHub CLI or a GitHub token)
  • File (creates issues as files in a local directory)
? What issue tracker would you like to use? (Use arrow keys)
❯ GitHub
  File

Configure GitHub Integration

If you chose GitHub as your issue tracker, you'll need to configure GitHub integration:

  • GitHub repository (owner/repo)
  • GitHub labels to apply to issues
  • Whether to use the GitHub CLI or REST API
  • GitHub token (if using REST API)
? What GitHub repository should Jouster use? TSavo/jouster
? What labels should Jouster apply to issues? bug,test-failure
? Should Jouster use the GitHub CLI or REST API? CLI

Configure Issue Templates

Jouster uses templates to generate GitHub issues for failing tests. You can customize these templates:

  • Use default templates
  • Customize templates
  • Specify a template directory
? Would you like to customize issue templates? No

Save Configuration

Finally, the wizard will save your configuration to a jouster.config.js file in your project root:

Configuration saved to jouster.config.js!

You can now run Jouster with:

$ npx jouster run-tests

Configuration File

The Setup Wizard generates a jouster.config.js file with your configuration. Here's an example:

// jouster.config.js
module.exports = {
  // Test runner configuration
  testRunnerType: 'jest',
  testRunnerOptions: {
    jest: {
      testPattern: '**/*.test.js',
      timeout: 5000,
      coverage: true
    }
  },
  
  // Issue tracker configuration
  trackerType: 'github',
  githubOptions: {
    repo: 'TSavo/jouster',
    labels: ['bug', 'test-failure'],
    useRest: false
  },
  
  // Template configuration
  templateDir: './node_modules/jouster/templates',
  
  // Database configuration
  databasePath: './test-issue-mapping.json'
};

Manual Configuration

If you prefer to configure Jouster manually, you can create a jouster.config.js file in your project root with the following structure:

// jouster.config.js
module.exports = {
  // Test runner configuration
  testRunnerType: 'jest', // 'jest', 'mocha', 'ava', 'tape', 'jasmine', 'vitest', 'cypress', 'playwright', 'testcafe', 'karma', 'generic'
  testRunnerOptions: {
    // Options specific to the chosen test runner
    jest: {
      // Jest-specific options
      testPattern: '**/*.test.js',
      timeout: 5000,
      coverage: true
    }
  },
  
  // Issue tracker configuration
  trackerType: 'github', // 'github' or 'file'
  githubOptions: {
    // GitHub-specific options (if trackerType is 'github')
    repo: 'owner/repo',
    labels: ['bug', 'test-failure'],
    useRest: false, // Whether to use the REST API instead of the CLI
    token: process.env.GITHUB_TOKEN // Required if useRest is true
  },
  fileOptions: {
    // File-specific options (if trackerType is 'file')
    bugsDir: './bugs'
  },
  
  // Template configuration
  templateDir: './templates', // Path to custom templates
  
  // Database configuration
  databasePath: './test-issue-mapping.json', // Path to the database file
  
  // Advanced options
  hooks: [], // Custom hooks
  plugins: [] // Custom plugins
};

Next Steps

After running the Setup Wizard, you can:

  • Run Jouster with npx jouster run-tests
  • Customize issue templates (see Templates)
  • Add custom hooks or plugins (see Advanced Features)
  • Configure CI integration (see Usage Guide)
